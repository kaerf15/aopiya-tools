# Workflow：交叉验证合稿（舆情 + 用户提供的经营数据）

在已完成 `sentiment-competitor.md` 之后，若用户**另外提供了**经营数据摘要（询盘、搜索、流量等，任意来源），用本文件做交叉验证并合稿。

无经营数据时只输出舆情简报，不必走本文件。

## 前置输入

| 输入 | 来源 |
|------|------|
| 舆情竞品简报 | `sentiment-competitor.md` 输出 |
| 经营数据摘要（可选） | 用户粘贴或口述：询盘趋势、市场/语种占比、GSC 词表现、渠道变化、内容转化等（不必固定格式） |

---

## 交叉验证

| 经营信号（用户提供） | 外部信号（本 Skill） | 综合解读 | 建议 |
|----------------------|----------------------|----------|------|
| 本期询盘 {↑↓}，{市场/语种} 占比变化 | 该市场是否有 P1+ 负面帖 | 是否可能抑制转化 | |
| 某搜索词展示↑点击↓ | 竞品在同词上的落地页/内容 | SEO 被抢还是 CTR 问题 | |
| AI/自然引荐来源变化 | 采购问句抽测是否引用本站 | 可见性是否一致 | |
| 某产品页转化偏弱 | 检索是否出现该产品差评 | 页面 vs 口碑 | |
| 品牌搜索需求变化 | 品牌负面提及量 | 需求 vs 声誉 | |
| 自然流量下滑 | 行业/竞品内容是否上新 | 竞争态势 | |

**原则**：

- 经营数据 **不因一条未核实帖下定论**
- 外部 P0/P1 **不因经营数据好就忽略**
- 建议：**可验证 + 可执行 + 落到 URL/市场/内容**

---

## 输出模板

```markdown
# 舆情与经营交叉分析报告（{period}）

## 执行摘要（3–5 句）

## 一、品牌舆情与竞品（外部）
## 二、经营数据摘要（用户提供）
## 三、交叉验证与综合判断
## 四、优先行动清单

## 附录
- 数据局限与未核实项
- 关键检索词与来源 URL
```

---

## 灵活偏离

- **时间紧**：保留 3 行交叉表 + 执行摘要
- **只有舆情数据**：跳过 §二，§三 仅记「待经营数据验证」
- **定向市场**：收窄检索与交叉表到单一市场
