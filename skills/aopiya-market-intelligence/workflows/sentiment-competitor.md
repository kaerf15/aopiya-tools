# Workflow：品牌舆情与竞品（外部调研）

**Web Search + 网页抓取/解析**。

背景与品牌分层见 `../references/brand-context.md`。

## 适用场景

| 场景 | 用本 workflow |
|------|----------------|
| 用户只要舆情/竞品简报 | 单独跑全文 |
| 用户另提供经营数据摘要 | 舆情跑完后接 `comprehensive-report.md` |
| 用户提及某市场经营异常 | 针对该市场/品牌做定向舆情检索 |

## 调研范围

### 监测主体

| 类型 | 关键词（多语组合检索） |
|------|------------------------|
| 集团 | AOPIYA、奥比亚、Guangzhou Aopiya Leather |
| 品牌 | SUSEN、CHRISBELLA、BAGCO |
| 负面信号词 | reviews、complaint、scam、quality issue、delay、refund（+ 法语/西语/阿语市场词按需） |

### 来源优先级（先高后低）

1. **B2B 平台**：Alibaba、Made-in-China、Global Sources 店铺评价与询盘区
2. **社媒**：TikTok、Instagram、YouTube（开箱、批发商吐槽、合作晒单）
3. **论坛/问答**：Reddit、Quora、行业 FB 群公开帖
4. **新闻/工商**：仅在有明确负面报道时收录

每个结论必须带 **可点击 URL + 抓取日期**；无法访问的标注「检索摘要，未逐页核实」。

## 执行步骤

### 1. 划定周期与焦点（默认近 30 天）

用户指定则从其；否则：

- 时间：近 30 天提及为主，可补充「历史仍被索引」的常青差评
- 市场：西非 / 中东 / 东南亚 至少各 1 组检索（见 `references/brand-context.md`）
- 若用户给了具体事件（「某国客户投诉交期」）→ 收窄检索词

### 2. 检索（Web Search）

按主体 × 信号词分批检索，示例：

```text
"AOPIYA" OR "奥比亚" reviews OR complaint
"SUSEN" handbag wholesale review
"CHRISBELLA" bag manufacturer
site:alibaba.com AOPIYA OR SUSEN
site:tiktok.com CHRISBELLA bag
handbag manufacturer China OEM competitor
best handbag manufacturer China 2025
```

记录：标题、摘要、URL、日期（若有）、初步正/负/中性。

### 3. 逐页解析（Fetch / 浏览器）

对 **高影响** 条目（负面投诉、B2B 平台差评、竞品官网首页）：

- 抓取正文：核心指控/赞誉、是否可验证、是否针对我司或同名混淆
- 竞品页：产品线、MOQ、认证宣称、内容/blog、语种、询盘入口

不必全量爬；**Top 5 负面 + Top 3 竞品** 逐页即可，其余用检索摘要。

### 4. 归类与评级

**情感**：正面 / 中性 / 负面 / 无法判断

**负面严重度**：

| 级别 | 定义 | 建议 |
|------|------|------|
| P0 | 欺诈/安全/大面积质量索赔且可核实 | 24h 内业务确认是否需回应 |
| P1 | 可核实交期/沟通差评，多源或高曝光 | 周会通报 + 运营/客服核对 |
| P2 | 单条、旧帖、或主体混淆疑似 | 记录观察，不主动放大 |
| P3 | 检索噪声/无关同名 | 忽略 |

**竞品对比维度**（每条竞品填表）：

- 定位与产品线、MOQ/定制政策
- 多语言站、博客/案例频率
- 在 ChatGPT / Perplexity 问采购问题时是否被引用（手动抽测 2–3 个采购问句）
- 相对我方：强/平/弱 + 一句证据

### 5. 输出（单独舆情/竞品报告）

```markdown

## 执行摘要
- 整体舆情：{偏正/中性/需关注}
- 需跟进：{P0/P1 条数或「无」}
- 竞品一句话：{我方相对位势}

## 1. 品牌提及概览
| 主体 | 正面 | 中性 | 负面 | 主要来源 |
|------|------|------|------|----------|
| AOPIYA | | | | |
| SUSEN | | | | |
| … | | | | |

## 2. 负面与风险（按严重度）
### P0 / P1
| 日期 | 来源 | URL | 摘要 | 是否需业务回应 |
|------|------|-----|------|----------------|

### P2 / 观察项

## 3. 正面与可放大信号

## 4. 竞品快照（Top 3–5）
| 竞品 | 定位 | MOQ/定制 | 内容/SEO | AI 可见性抽测 | vs AOPIYA |
|------|------|----------|----------|---------------|-----------|

## 5. 建议动作（≤5 条，可执行）

## 6. 数据局限
- 外部检索覆盖不全；未登录平台可能看不到全部评价
- 与经营转化无直接因果；用户另提供经营数据时可接 `comprehensive-report.md`
```

## 灵活偏离

- **只做舆情不做竞品**：跳过竞品问句与 §4
- **只做竞品**：检索竞品词 + AI 抽测，§2 仅记及我司提及
- **单市场深挖**：检索词改为该国语言 + 当地 B2B/社媒

## 禁止

- 把检索摘要当事实而不标注来源 URL
- 将未核实的负面当 P0 通报
- 用舆情点击/曝光冒充站内流量指标
